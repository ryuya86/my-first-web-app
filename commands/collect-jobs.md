# タスク: 新着案件の収集とスコアリング

## 目的
CrowdWorksのRSSフィードから新着案件を収集し、マッチングスコアを算出し、高スコア案件をSlackに通知する。

## 手順

### Step 1: RSS案件取得
`python tools/rss_collector.py` を実行する。
結果はJSON配列で標準出力に出力される。新着案件がなければ空配列`[]`が返る。

新着なしの場合はSlackに「新着案件なし」と報告して終了する。

### Step 2: マッチングスコアリング
各案件について、以下の基準で0〜100点のスコアを算出する。
`templates/crowdworks-profile.md` のスキルセットを参照し、自分の判断でスコアをつける。

**現在のフェーズ: 実績ゼロ〜数件（実績作り最優先）**
報酬よりも「受注できるか」「スキルに合うか」を最重視する。

**評価基準（合計100点）:**

1. **受注しやすさ (35点)** — 最重視
   - 応募者数が少ない（num_applicationsが低い）ほど高得点
   - 要件がシンプルで初心者でも対応可能なら高得点
   - 「未経験OK」「初心者歓迎」の記載があれば加点
   - 納期に余裕があれば加点
   - 応募者数20人以上 → 大幅減点（競争が激しい）

2. **スキルマッチ (30点)** — 重視
   CEOのポートフォリオ（実績あり）を参照してマッチ度を評価:

   **最高マッチ（25-30点）— ポートフォリオに実績がある分野:**
   - GAS × API連携（スカウトメッセージ自動生成システム構築実績あり）
   - Gmail API（返信下書き自動生成ツール開発実績あり）
   - Slack API / bot開発（自動返信bot開発・現在も稼働中）
   - Python / Node.js スクレイピング → スプレッドシート出力（データ収集自動化実績あり）

   **高マッチ（15-24点）— 関連スキルで対応可能:**
   - スプレッドシート自動化 / Excel VBA / データ整理
   - API連携全般 / Webhook / 自動化ツール開発
   - データ入力 / リスト作成 / CSV処理

   **中マッチ（5-14点）— 対応可能だが実績なし:**
   - WordPress / LP制作 / HTML/CSS
   - 業務効率化コンサル

   **不一致（0点）:**
   - 撮影 / デザイン / 動画編集 / ライティング / テレアポ / インタビュー / 翻訳
   → スキルが全く合わない案件は即スキップ

3. **要件の明確さ (20点)**
   - 成果物・納期・報酬が明確に記載されている → 高得点
   - 「詳細は面談で」のみ → 減点（ただし実績作りフェーズなので0点にはしない）
   - 作業手順まで書いてある → 加点

4. **リスクの低さ (15点)**
   - クライアントの評価が高い / 発注実績が多い → 加点
   - 「至急」「今日中」など非現実的な納期 → 大幅減点
   - 報酬が相場より極端に低い（搾取リスク） → 減点
   - 要件が曖昧すぎて範囲が際限なく広がりそう → 減点

**スコア閾値:**
- 60点以上: Slack通知（提案文付き）
- 40〜59点: Slack通知（概要のみ、提案文なし）
- 40点未満: スキップ

### Step 3: 提案文生成
スコア60点以上の案件について、提案文を生成する。
`templates/proposal-guidelines.md` のガイドラインに従うこと。

提案文を生成したら、`python tools/ng_checker.py '提案文テキスト'` でNGチェックを通す。
NGが検出されたら修正してから次に進む。

スコア40〜59点の案件は提案文なしで概要のみSlackに通知する。

### Step 4: DB記録
各案件を `python tools/db.py log-job '{json}'` で記録する。
JSONには `id`, `title`, `url`, `category`, `score`, `proposal` を含める。

### Step 5: Slack通知
各案件を `python tools/slack_notify.py job_found '{json}'` で通知する。
JSONには `title`, `url`, `score`, `category`, `summary`, `proposal` を含める。

### Step 6: 完了報告
全案件の処理が終わったら、処理結果のサマリーをSlackに報告する:
```
python tools/slack_notify.py report '{"title":"案件収集完了","body":"新着: X件 / スコア通過: Y件 / スキップ: Z件"}'
```

## 注意事項
- 案件一覧をまず全件ざっと見て、明らかにスキルに合わない案件（撮影、ダンサー、インタビュー、テレアポ等）はスコアリングせずスキップする
- スコアリング対象は最大20件まで（タイトルと概要から判断して有望な順に選ぶ）
- 提案文生成は最大5件まで（スコアが高い順）
- エラーが発生しても他の案件の処理は続行する
- 全体でエラーがあった場合は最後にまとめてSlackに報告する
