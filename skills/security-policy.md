# Security Policy

## 目的
クライアント情報・個人情報・機密データを適切に管理し、情報漏洩リスクを最小化する。

## ルール
1. **機密レベル分類**：全情報を3段階に分類して扱う
   - **レベルA（最機密）**：契約書・見積書・クライアント個人情報 → 暗号化ストレージのみ、共有禁止
   - **レベルB（社内限定）**：プロジェクト進捗・内部資料 → 社内メンバーのみ閲覧可
   - **レベルC（共有可）**：公開資料・マーケティングコンテンツ → 外部共有可
2. **NDA遵守**：クライアントと締結したNDAの内容を業務開始前に確認し、対象情報を第三者に開示しない
3. **個人情報保護**：氏名・メールアドレス・電話番号等の個人情報はレベルA扱いとし、業務目的以外で使用しない
4. **パスワード管理**：パスワードの平文記録・メール送信を禁止する。パスワードマネージャーを使用すること
5. **AIツールへの入力制限**：レベルAの情報をClaude・ChatGPT等の外部AIサービスに入力することを禁止する
6. **インシデント対応**：情報漏洩の疑いが生じた場合は即座にCEOに報告し、対象クライアントへの通知を24時間以内に行う

## 適用例
- クライアントの事業計画書（レベルA）をNotionに保存する際は、アクセス権限をCEOのみに設定する
- 新しいパートナーへの業務委託前に必ずNDAを締結し、共有情報の範囲を書面で確認する
- クライアントのメールアドレスをAIプロンプトにそのまま貼り付けず、「担当者A」などに置換して使用する
