# Tools Manual

## 目的
業務で使用するツールの用途・使い分けルールを定め、ツール選択の迷いと二重管理を防ぐ。

## ルール
1. **Claude Code**：コード生成・レビュー・ドキュメント自動作成・スキル実行に使用する。AIワークフローの中核ツール
2. **GitHub**：ソースコード管理・バージョン管理・CI/CDパイプライン管理に使用する。コードはすべてGitHubで一元管理する
3. **Notion**：社内ナレッジベース・プロジェクト管理・議事録・スキルファイル管理に使用する。ドキュメントの単一ソース・オブ・トゥルース
4. **Google Workspace**：クライアントとの共有ドキュメント（Docs・Sheets・Slides）・メール（Gmail）・カレンダー管理に使用する
5. **X（Twitter）**：リードジェネレーション・ブランディング・採用に使用する。週3投稿以上を維持する
6. **YouTube Studio**：教育コンテンツ・事例紹介動画の公開・アナリティクス管理に使用する
7. **LINE公式アカウント**：既存クライアントへのリマインド・キャンペーン通知・FAQ対応自動化に使用する
8. **Stripe**：請求書発行・定期課金（リテイナー）・支払い管理に使用する。手動集金は原則禁止

## 適用例
- クライアントへの提案書はGoogle Slidesで作成し、NotionにリンクとPDFを保管する
- 新機能のコード変更はGitHubのPRで管理し、Claude Codeでレビューを補助する
- 月次リテイナー請求はStripeで自動化し、入金確認をNotionのダッシュボードで一元管理する
