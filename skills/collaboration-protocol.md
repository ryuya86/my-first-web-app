# Collaboration Protocol

## 目的
仮想チーム内の部門間・エージェント間の連携ルールを定め、ハンドオフの抜け漏れと二重作業を防ぐ。

## ルール
1. **Lead/Support体制**：各タスクには必ず1名のLead（主担当）と必要に応じてSupport（支援担当）を設定する。最終責任はLeadが持つ
2. **ハンドオフルール**：タスクを次の担当者に引き継ぐ際は「完了内容・残課題・次のアクション・期限」を明記したハンドオフメモをNotionに残す
3. **依存関係の明示**：自分のタスクが他のタスクの完了を待っている場合、Notionのタスクに依存関係を設定しLeadに通知する
4. **統合ルール**：複数担当者が作成した成果物を統合する際は、Leadが最終版をレビューし矛盾・重複を除去してからクライアントに提出する
5. **コミュニケーション頻度**：プロジェクト進行中は最低週1回の同期（非同期メモでも可）を行い、ブロッカーを早期に共有する
6. **意思決定ログ**：重要な判断（スコープ変更・技術選定など）はNotionの議事録に記録し、後から経緯を追えるようにする

## 適用例
- 提案書作成：営業担当がLeadとなり、技術担当がSupport（技術仕様のみ担当）。最終レビューはLeadが実施する
- 納品物の引き継ぎ：開発担当から納品担当へのハンドオフ時に「テスト済み機能・既知バグ・クライアント要望メモ」を添付する
- 複数エージェントが同時作業する場合、各エージェントの作業範囲をNotionのタスク分割で明確化する
