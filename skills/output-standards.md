# Output Standards

## 目的
全成果物（提案書・レポート・メール・資料）の品質基準を統一し、手戻りをゼロに近づける。

## ルール
1. **エグゼクティブサマリー必須**：全成果物の冒頭に3〜5行の要約を置く。読者が本文を読まずとも意図が伝わること
2. **数値・期限を含める**：主張・提案・計画には必ず具体的な数値（金額・工数・件数など）と期限を明記する
3. **ネクストアクション明記**：各成果物の末尾に「誰が・何を・いつまでに」を箇条書きで記載する
4. **構成の統一**：サマリー → 背景・課題 → 提案・詳細 → ネクストアクション の順番を守る
5. **提出前チェック**：「数値の根拠はあるか」「期限は明記されているか」「アクションは具体的か」を自己確認する
6. **ファイル命名規則**：`YYYYMMDD_案件名_資料種別.拡張子`（例：`20260322_株式会社A_提案書.pdf`）
7. **バージョン管理**：修正が発生した場合はファイル名末尾に`_v2`などを付与し、旧版を上書きしない

## 適用例
- 提案書作成時：冒頭に「本提案で解決する課題・期間・概算費用」を3行でまとめる
- 週次レポート：「今週のKPI達成状況（数値）→ 課題 → 来週のアクション」の構成で作成する
- メール返信：結論を1文目に書き、補足説明は2段落目以降に置く
