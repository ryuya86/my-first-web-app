# CrowdWorks プロフィール設計書

## 基本方針

- **ポジショニング**: 「手が回らない」を丸ごと引き受ける、AI活用の何でも屋
- **トーン**: 発注者の不安に寄り添う。「この人なら安心」と思わせる誠実さ
- **構成原則**: 機能（何ができる）ではなくベネフィット（どう良くなる）で語る
- **NG**: 根拠なし表現（激安、圧倒的、業界No.1）、過度な自己アピール

## カテゴリ・受注スタイル設定

### 希望カテゴリ（推奨）
1. **事務・カンタン作業** → 初期実績を積みやすい
2. **Web制作・Webデザイン** → 単価が上がりやすい
3. **システム開発・アプリ開発** → 専門性アピール
4. **ライティング・記事作成** → 案件数が多い

### 仕事の形式
- **タスク**: 最初の実績作りに（簡単なデータ入力等）
- **プロジェクト（固定報酬）**: メインの受注形式
- **プロジェクト（時間単価）**: 継続案件に

---

## 新規アカウント 実績構築ロードマップ

### Phase 1: 信頼の種まき（最初の1〜2週間）
- タスク案件を5〜10件こなして評価を貯める
- プロフィール完成度100%にする（写真・本人確認・スキルテスト）
- 低単価でも丁寧に対応し、★5評価を獲得

### Phase 2: プロジェクト案件に挑戦（2〜4週間目）
- データ入力・リサーチ系のプロジェクト案件に応募
- 提案文は案件ごとにカスタマイズ（テンプレ感を出さない）
- 1件でも受注→高評価を得る

### Phase 3: 単価アップ＆専門案件へ（1ヶ月〜）
- 評価4.5以上 + 受注実績5件以上を達成
- 自動化・スクレイピング等の専門案件にシフト
- リピーター獲得を狙う

## 提案文との連携

このプロフィールは `tools/crowdworks/proposal_generator.py` の提案生成と
トーンを統一しています：

- プロフィール → 「何ができる人か」を網羅的に伝える
- 提案文 → 「この案件に対して具体的に何ができるか」を伝える

プロフィールを見た発注者が「この人なら任せられそう」と思い、
提案文で「この案件をちゃんと理解している」と確信する、
2段階の信頼構築を設計しています。
